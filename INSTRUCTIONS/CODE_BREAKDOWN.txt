FILE: lib/providers/app_state.dart, Lines 400-520

The _detectConflicts() method is the core of the system.
It finds all scheduling problems and creates ScheduleConflict objects.

Pseudocode Flow:

  _detectConflicts() {
    List<ScheduleConflict> newConflicts = [];
    
    // For each scheduled class:
    for (ScheduleEntry entry in _scheduleEntries) {
      
      // 1. CHECK: Does this room have an override?
      for (RoomOverride override in _roomOverrides) {
        if (override.roomId == entry.room.id) {
          if (override.coversDate(entry.day)) {
            // CONFLICT: Room is blocked
            newConflicts.add(ScheduleConflict(
              type: ConflictType.roomBlockedByOverride,
              description: "Room blocked: ${override.reason}"
            ));
          }
        }
      }
      
      // 2. CHECK: Is teacher teaching two classes at same time?
      for (ScheduleEntry other in _scheduleEntries) {
        if (other.id != entry.id) {  // Different class
          if (other.teacher.id == entry.teacher.id) {  // Same teacher
            if (other.day == entry.day) {  // Same day
              if (timesOverlap(entry.timeStart, entry.timeEnd,
                               other.timeStart, other.timeEnd)) {
                // CONFLICT: Teacher double-scheduled
                newConflicts.add(ScheduleConflict(
                  type: ConflictType.teacherDoubleScheduled,
                  conflictingEntry1: entry,
                  conflictingEntry2: other,
                  description: "Teacher scheduled for two classes"
                ));
              }
            }
          }
        }
      }
      
      // 3. CHECK: Is room booked twice at same time?
      for (ScheduleEntry other in _scheduleEntries) {
        if (other.id != entry.id) {
          if (other.room.id == entry.room.id) {  // Same room
            if (other.day == entry.day) {  // Same day
              if (timesOverlap(entry.timeStart, entry.timeEnd,
                               other.timeStart, other.timeEnd)) {
                // CONFLICT: Room double-booked
                newConflicts.add(ScheduleConflict(
                  type: ConflictType.doubleBookedRoom,
                  ...
                ));
              }
            }
          }
        }
      }
      
      // 4. CHECK: Does room have required equipment?
      if (entry.subject.requiresProjector && !entry.room.hasProjector) {
        // CONFLICT: No projector
        newConflicts.add(...);
      }
      
      if (entry.subject.requiresComputers && !entry.room.hasComputers) {
        // CONFLICT: No computers
        newConflicts.add(...);
      }
      
      // 5. CHECK: Does room match required type?
      if (entry.subject.type == SubjectType.laboratory) {
        if (entry.room.type != RoomType.laboratory) {
          // CONFLICT: Lab subject needs lab room
          newConflicts.add(...);
        }
      }
      
      // 6. CHECK: Is room large enough?
      int sectionSize = calculateEnrollment(entry.section);
      if (sectionSize > entry.room.capacity) {
        // CONFLICT: Too many students for room
        newConflicts.add(...);
      }
    }
    
    _conflicts = newConflicts;
    notifyListeners();
  }

KEY ALGORITHM DETAILS:

1. Nested Loops
   - Outer: For each schedule entry
   - Inner: Compare with every other entry/override/etc
   - Complexity: O(n²) - can be slow with many entries
   - Optimization: Could use indexes/hashing

2. Time Overlap Check
   The system needs a helper method to detect overlapping times:
   
   bool timesOverlap(String start1, String end1,
                     String start2, String end2) {
     // Convert "HH:mm" strings to minutes since midnight
     int s1 = timeToMinutes(start1);  // e.g. "08:00" → 480
     int e1 = timeToMinutes(end1);    // e.g. "09:30" → 570
     int s2 = timeToMinutes(start2);
     int e2 = timeToMinutes(end2);
     
     // Overlap if: s1 < e2 AND s2 < e1
     return s1 < e2 && s2 < e1;
   }
   
   Examples:
     Class 1: 08:00-09:30
     Class 2: 08:30-10:00
     Overlap: 08:30-09:30 ✓ CONFLICT
     
     Class 1: 08:00-09:30
     Class 2: 09:30-11:00
     Overlap: None (back-to-back) ✓ OK
     
     Class 1: 08:00-09:30
     Class 2: 14:00-15:30
     Overlap: None (different time) ✓ OK

3. RoomOverride Impact
   
   When override is created:
     RoomOverride {
       roomId: "r1",
       roomName: "1A",
       reason: "Faculty Meeting",
       startDate: Jan 15,
       endDate: Jan 15
     }
   
   During _detectConflicts():
     If any schedule entry:
       - Is in room "r1"
       - Is on Jan 15
     Then:
       Create conflict: "Room 1A blocked due to Faculty Meeting"
   
   This prevents scheduling (conflicts resolved by human admin)
